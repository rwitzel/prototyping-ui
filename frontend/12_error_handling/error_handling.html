Error Handling

Debug Output

Print helpful information via console.log('Ooops!',var1,var2). Separate the arguments with comma so that you can explore them better.

Debugger

Add the debugger;keyword in the code. The browser considers this a breakpoint.

Scope Availability

Use a global variable to examine variables that are kept private by AngularJS, like $scope. In the code base here the variable debug_context is used for that purpose.

Alert

Use alert('Something went wrong') to show an error in an modal dialog.

Error visualization

Write the state ('In progress', 'Finished', 'Failed with error ...') in an text box near the button that triggered the action. Example: The result area in section Backend Calls.
